How DNS Works

A step-by-step explanation of how your computer finds the correct IP address for any website.

What Happens When You Type a Website Into Your Browser?

When you enter a domain name like google.com into your browser, your device begins a DNS lookup. This process finds the correct IP address so your browser knows where to send the request. DNS works behind the scenes in milliseconds, but several important steps happen along the way.

DNS works so quickly that most users never notice it — but without it, the internet would be nearly impossible to use.

Step-by-Step: How a DNS Lookup Works

  1. Your browser checks its cache. If it has recently visited the site, it may already know the IP address.
  2. Your operating system checks its cache. If the browser doesn’t know, your computer might.
  3. The recursive resolver is contacted. This is usually run by your ISP or a public DNS provider like Google or Cloudflare.
  4. The resolver checks its own cache. If it has the answer, the lookup ends here.
  5. The resolver asks a root server. The root server directs the resolver to the correct top-level domain (TLD) server.
  6. The resolver asks the TLD server. For example, the .com TLD server points to the authoritative server for google.com.
  7. The resolver asks the authoritative server. This server holds the official DNS records and returns the IP address.
  8. The resolver sends the IP address back to your device.
  9. Your browser connects to the website.

Even though this process involves multiple servers, it usually completes in under a hundred milliseconds thanks to caching and optimized routing.

Why DNS Is So Fast

DNS is designed for speed. Most DNS lookups take only a few milliseconds because recursive resolvers maintain large caches and use optimized routing. Many DNS providers also use Anycast networks, which route your request to the closest available server.

  • Caching: Saves previous lookups to avoid repeating work.
  • Anycast routing: Sends your request to the nearest DNS server.
  • Distributed infrastructure: DNS servers exist all over the world.
  • Lightweight protocol: DNS messages are small and efficient.

This combination ensures that DNS rarely becomes a bottleneck, even during high traffic periods.
