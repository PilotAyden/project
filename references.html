References

Below is a list of sources used throughout this project, each with a brief description of its relevance.

Works Cited

  • “Authoritative vs. Recursive DNS Explained” – DNSFilter
    https://www.dnsfilter.com/resources/authoritative-vs-recursive-dns
    This article clearly explains the difference between authoritative and recursive DNS servers, which helped shape the Key Concepts and How DNS Works pages.
  • “BIND Explained: A Powerful Tool for DNS Management” – ClouDNS Blog
    https://www.cloudns.net/blog/bind-explained/
    Provided background on BIND, one of the most widely used DNS server software packages, useful for understanding DNS infrastructure.
  • “DNS History: When and Why Was DNS Created?” – ClouDNS Blog
    https://www.cloudns.net/blog/dns-history/
    Helped build the History page by explaining the origins of DNS and the problems it solved.
  • “DNS TTL Values: Tutorial & Best Practices” – ClouDNS
    https://www.cloudns.net/blog/dns-ttl-values/
    Used to explain caching behavior and TTL values in the How DNS Works and Key Concepts pages.
  • “Introduction to Google Public DNS” – Google Developers
    https://developers.google.com/speed/public-dns/docs/intro
    Provided details about recursive resolvers and how public DNS services like Google’s operate.
  • “The Future of DNS: Trends and Technologies Shaping the Internet” – CheckDNS
    https://checkdns.net/blog/future-of-dns/
    Helped expand the Modern DNS Challenges section with information on new technologies and security improvements.
  • “The Next Decade of DNS: Predictions and Emerging Trends” – DN.org
    https://dn.org/dns-future-trends/
    Provided insight into long‑term DNS evolution, including encrypted DNS and global infrastructure changes.
  • “What Is a DNS Server and How Does It Function?” – Cyberly
    https://cyber.ly/what-is-dns-server/
    Used for foundational explanations of DNS servers and their roles in the lookup process.
  • “What Is DNS? | How DNS Works” – Cloudflare
    https://www.cloudflare.com/learning/dns/what-is-dns/
    One of the primary sources for explaining DNS basics, caching, and resolver behavior.
  • “What Is a Top-Level Domain (TLD)?” – Cloudflare
    https://www.cloudflare.com/learning/dns/top-level-domain/
    Provided definitions and examples of TLDs used in the Key Concepts and History pages.
  • RFC 1034 & RFC 1035 – Domain Names: Concepts and Facilities
    https://www.rfc-editor.org/rfc/rfc1034
    https://www.rfc-editor.org/rfc/rfc1035
    These foundational RFCs define how DNS works at a technical level and were used to verify accuracy in multiple sections.
