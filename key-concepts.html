Key Concepts it the Domain Name System (DNS)

Here are some important terms that help explain how DNS Works

1. DNS (Domain Name System)

DNS is the system that translates human-readable domain names like google.com into numerical IP addresses that computers use to communicate.

2. Recursive Resolver

The recursive resolver is the server that does the searching for you. When you type a website into your browser, the resolver asks other DNS servers until it finds the correct IP address.

3. Authoritative DNS Server

This server holds the official DNS records for a domain. It provides the final answer in the lookup process, such as the IP address for a website.

4. Root and TLD servers

Root servers direct DNS queries to the correct top-level domain (TLD) servers, such as .com, .org, or .edu. TLD servers then point the resolver to the correct authoritative server.

5. DNS Caching

DNS caching stores previous lookup results so that the resolver does not have to repeat the entire process. This makes browsing faster and reduces network traffic

6. Anycast Routing

Anycast is a routing method where multiple servers share the same IP address. DNS uses Anycast to send users to the closest and fastest server available.

7. DNSSEC (DNS Security Extensions)

DNSSEC adds cryptographic signatures to DNS records to protect users from tampered or fake DNS responses. It improves security but is not fully adopted everywhere yet.
