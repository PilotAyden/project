Key Concepts in the Domain Name System (DNS)

These core ideas form the foundation of how DNS works behind the scenes to keep the internet running smoothly.

DNS Hierarchy

DNS is organized as a hierarchical tree structure. At the top is the root, followed by top-level domains (TLDs) like .com, .org, and .edu. Below TLDs are second-level domains such as example.com, and beneath those are subdomains.

This hierarchy allows DNS to scale globally while delegating responsibility to different organizations.

Root Servers

Root servers are the highest level of DNS. They don’t store IP addresses for websites, but they direct resolvers to the correct TLD name servers. There are 13 root server clusters worldwide, each operated by different organizations.

Even though there are only 13 root server names, each one is backed by hundreds of physical servers using Anycast.

Top-Level Domains (TLDs)

TLDs are the first level beneath the root. Examples include .com, .org, .net, and country codes like .uk or .jp. Each TLD is managed by a registry that maintains authoritative DNS servers for that domain.

Authoritative Name Servers

Authoritative servers store the official DNS records for a domain. When a resolver needs the IP address for a website, the authoritative server provides the final answer. Organizations often run their own authoritative servers or use DNS hosting providers.

Recursive Resolvers

A recursive resolver performs the full DNS lookup on behalf of the user. It checks its cache, queries root servers, TLD servers, and authoritative servers as needed. ISPs, Google (8.8.8.8), and Cloudflare (1.1.1.1) operate popular recursive resolvers.

DNS Records

DNS records define how a domain behaves. Common types include:

  • A: IPv4 address
  • AAAA: IPv6 address
  • CNAME: Alias to another domain
  • MX: Mail server information
  • NS: Name server for the domain
  • TXT: Text data for verification and security

DNS Caching

DNS caching stores lookup results temporarily to speed up future requests. Caching happens at multiple layers: your browser, your operating system, and your recursive resolver. Each record has a TTL (Time To Live) that determines how long it stays cached.

DNSSEC

DNSSEC adds cryptographic signatures to DNS responses, preventing attackers from forging or tampering with DNS data. It ensures authenticity but does not encrypt the data itself.

Encrypted DNS (DoH & DoT)

Modern DNS protocols like DNS over HTTPS (DoH) and DNS over TLS (DoT) encrypt DNS queries to protect user privacy. These protocols prevent ISPs and attackers from seeing which websites you are looking up.
